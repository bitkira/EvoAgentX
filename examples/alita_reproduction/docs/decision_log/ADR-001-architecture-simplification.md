# ADR-001: 架构简化决策

## 状态
**接受** - 2025-08-12

## 背景
基于两位顶级AI专家(Gemini 2.5 Pro和OpenAI o3)的咨询建议，对ALITA项目的22个commit开发计划进行架构优化，以降低技术风险并提高开发效率。

## 决策

### 1. Phase 1架构简化
**决策**: 将Commit 4从"创建独立WebAgent"改为"WebSearch工具集成"

**理由**:
- 过早引入多智能体协作会增加系统复杂度
- 早期应专注于核心功能实现
- 多智能体协作的调试和维护成本较高

**影响**:
- Commit 4开发时间从2-3天缩短到1-2天
- 降低Phase 1的技术风险
- 为Phase 3的多智能体重构预留清晰接口

### 2. Docker集成提前
**决策**: 将DockerInterpreterToolkit集成从Commit 13提前到Commit 6

**理由**:
- 安全执行是脚本执行的核心风险点
- 避免后期从自建sandbox到Docker的重构成本
- EvoAgentX框架提供现成的Docker工具包

**影响**:
- 从源头解决代码安全执行问题
- 减少技术债务
- 提高系统安全性

### 3. 存储系统优化
**决策**: MCPBox基于EvoAgentX的StorageHandler实现，而非文件系统

**理由**:
- 避免重复造轮子
- 获得多后端支持(SQLite/MongoDB)
- 便于未来扩展和迁移

**影响**:
- 提高存储系统的可扩展性
- 减少维护复杂度
- 与框架生态深度集成

## 替代方案

### 方案A: 保持原始计划
- 优点: 忠实复现原始ALITA架构
- 缺点: 技术风险高，开发复杂度大

### 方案B: 完全重写架构
- 优点: 可以设计最优架构
- 缺点: 偏离项目目标，时间成本高

**选择决策**: 采用专家建议的优化方案，平衡复现准确性与工程实用性

## 后果

### 正面影响
- **风险降低**: 技术风险更可控
- **开发加速**: Phase 1可能提前完成
- **质量提升**: 基于成熟框架组件
- **维护友好**: 减少技术债务

### 负面影响
- **架构偏差**: 与原始ALITA设计有所不同
- **学习成本**: 需要深度理解EvoAgentX框架
- **依赖风险**: 更依赖框架稳定性

### 缓解措施
- 详细文档记录架构变更原因
- 在关键节点验证与原始ALITA的功能对等性
- 保持核心业务逻辑的独立性

## 合规性
此决策符合以下原则:
- 技术风险最小化
- 开发效率最大化
- 代码质量保证
- 长期维护性考虑
